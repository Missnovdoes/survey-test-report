## 二、界面测试

### 测什么
| 检查项 | 说明 |
|--------|------|
| 图片显示 | 所有图标、Logo、banner 正确加载，无裂图 |
| 文字内容 | 无错别字、无乱码、无截断 |
| 布局对齐 | 表单、按钮、表格排列整齐，无错位 |
| 提示信息 | 错误提示准确，位置不遮挡关键内容 |

### 怎么做
1. 逐页截图关键界面（登录/注册/主页/题库/用户管理/个人设置）
2. 标注检查结果，有问题的截图标红
3. 一般界面测试不单独建章节，附在兼容性测试后面或作为一段总结

### 预计时间：10 分钟

## 三、自动化测试

### 整体架构
- 框架：Selenium 4.x + Java（Maven）
- 浏览器驱动：WebDriverManager 自动管理
- 截图工具：Apache Commons IO
- 设计模式：Page Object —— 每个页面一个类，公共方法抽到 Utils

### 项目结构
```
survey-autotest/
├── pom.xml
├── src/test/java/
│   ├── common/
│   │   └── Utils.java          # WebDriver 初始化、截图、销毁
│   ├── tests/
│   │   ├── RegisterTest.java   # 注册 5 场景
│   │   ├── LoginTest.java      # 登录 6 场景
│   │   ├── HomeTest.java       # 首页 4 场景
│   │   ├── ProjectTest.java    # 我的项目 3 场景
│   │   ├── RepoTest.java       # 我的题库 3 场景
│   │   ├── TemplateTest.java   # 模板广场 2 场景
│   │   └── UserSettingTest.java # 用户管理 + 个人设置
│   └── image/                  # 自动截图输出目录
```

### 覆盖范围
| 模块 | 测试类 | 自动化用例数 | 验证方式 |
|------|-------|------------|---------|
| 注册 | RegisterTest | 5 | assert 页面跳转 / 错误提示文案 |
| 登录 | LoginTest | 6 | assert 页面标题 / 错误提示 |
| 首页 | HomeTest | 4 | assert 页面元素存在 + 跳转 |
| 我的项目 | ProjectTest | 3 | assert 项目列表元素 |
| 我的题库 | RepoTest | 3 | assert 创建成功后列表更新 |
| 模板广场 | TemplateTest | 2 | assert 搜索结果 |
| 用户管理+设置 | UserSettingTest | 4 | assert 创建成功 / 密码修改提示 |
| **合计** | **7 模块** | **27** | |

### 预期结果
- 自动化用例 27 条
- 手工无法通过的用例（BUG-01 等）自动化同样失败，保持一致
- 自动化通过率：预计与手工测试一致（约 85%）

### 怎么做
1. 创建 Maven 项目，配置 pom.xml（selenium-java 4.x + webdrivermanager 5.x）
2. 写 Utils.java 公共类
3. 逐个模块写测试类
4. 运行 → 截图输出到 image/ 目录
5. 把代码贴到报告「七、自动化测试」章节

### 预计时间：2-3 小时（有 CSDN 参考可大幅缩短）

## 四、性能测试

### 测试目标
验证问卷考试系统在单用户并发下的响应时间、吞吐量和稳定性。

### 工具
- **JMeter 5.5**：压力测试
- **Fiddler**：抓包验证请求明文/密文（安全性检查）

### 测试接口清单
| 序号 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 1 | /api/public/login | POST | 登录 |
| 2 | /api/public/register | POST | 注册 |
| 3 | /api/project/list | GET | 项目列表 |
| 4 | /api/repo/index | GET | 题库列表 |
| 5 | /api/template | GET | 模板广场 |
| 6 | /api/system/user | GET | 用户管理 |
| 7 | /api/system/setting | GET | 个人设置 |
| 8 | /api/repo/question | GET | 问题管理 |

### JMeter 配置
- 线程数：10（模拟 10 个并发用户）
- Ramp-Up 时间：5 秒
- 循环次数：5 次
- 监听器：聚合报告 + 查看结果树 + 汇总报告

### 预期输出指标
| 指标 | 说明 |
|------|------|
| 平均响应时间 | 期望 < 500ms |
| 吞吐量 (TPS) | 期望 > 1 req/s |
| 错误率 | 期望 0% |
| 90% 响应时间 | 期望 < 1000ms |
| 峰值响应时间 | 记录最大值 |

### Fiddler 抓包检查
- 登录接口：检查密码是否明文传输
- 结论：如为密文 → 安全性合格；如为明文 → 记录为安全风险

### 怎么做
1. 打开 JMeter → 新建测试计划
2. 添加线程组（10 线程 × 5 循环）
3. 添加 HTTP 请求默认值（服务器 IP: 49.235.61.184，端口: 8080）
4. 为每个接口添加 HTTP 请求采样器
5. 添加监听器（聚合报告、汇总报告）
6. 运行 → 导出报告
7. Fiddler 打开 → 登录一次 → 检查请求体中密码字段

### 预计时间：1 小时
