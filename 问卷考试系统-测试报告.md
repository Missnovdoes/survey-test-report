#### 4.3.3 我的项目页面

① 进入"我的项目"页面

预期结果：展示项目列表。

实际结果：页面内容空白，无任何数据。❌ **失败 — BUG-01**

![我的项目-空白](screenshots/project-01-list-blank.png)

![我的项目-页面空白](screenshots/project-02-page-blank.png)

**Network 抓包分析**：

通过 Chrome DevTools 抓包验证：

- **创建问卷接口** `POST /api/project/create` → `code: 200`，项目 ID: E8nHOe，数据完整：

```json
{
    "code": 200,
    "data": {
        "id": "E8nHOe",
        "name": "请输入标题",
        "mode": "survey",
        "survey": {
            "id": "E8nHOe",
            "title": "请输入标题",
            "children": [{ "id": "atj5", "title": "单行文本题", "type": "FillBlank" }]
        },
        "setting": { "status": 0 },
        "createAt": "2026-06-22 13:10:51"
    }
}
```

- **项目列表接口** `GET /api/project/list?current=1&mode=folder&pageSize=1024` → `total: 0, list: []`：

```json
{ "code": 200, "data": { "total": 0, "list": [] } }
```

**Bug 分析**：创建接口明确返回了完整的项目数据（ID: E8nHOe, mode: survey），但列表接口使用 `mode=folder` 作为默认查询参数，与项目实际的 `mode=survey` 不匹配，导致查询结果为空。系统在前端展示时未做正确的模式切换，页面白屏无任何提示。

② 搜索框输入项目名称查询 & 回收站功能测试

预期结果：筛选匹配的项目 / 弹出确认提示框。

实际结果：因列表无数据，无法验证上述功能。⏭ 阻塞

> **我的项目模块小结**：6 条用例中 2 条通过，1 条失败（BUG-01/BUG-02），3 条因 Bug 阻塞。通过率 33.3%。此 Bug 严重阻塞问卷管理→发布→收集→统计全业务流程。
